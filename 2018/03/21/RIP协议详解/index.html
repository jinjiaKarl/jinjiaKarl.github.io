RIP协议详解

1.动态路由协议
作用：动态路由协议是路由器间交换网络信息，计算路径的机制。
AS：自制系统，是由若干拥有统一管理策略和维护策略所组成的，其实就是一个网络。AS-id：1-65535。
AS内动态路由协议（IGPS内部网关协议集）：RIP、EIGRP、OSPF、IS-IS
AS间动态路由协议（EGPS边界网关协议集）：BGP

2.RIP协议：距离矢量路由协议
RIPv1、RIPv2适用IPv4，RIPng适用IPv6。
RIP路由代码R，管理距离AD：120
3.RIPv1配置（全局配置模式下）
router rip //启用rip
version 1 //设置版本
network //宣告网络
宣告的作用：将主网包含的子网变成更新、将子网对应的物理接口划入rip发送更新。
只能主网宣告，是有类的路由协议
4.RIPv1详解
1）更新方式：UDP广播更新，目的IP：255.255.255.255，目的端口号：520，源端口号随机，更新包携带25个条目，周期性更新30s。
2）度量值metric计算：更新发送时，跳数hop+1，最大跳数为15。
3）环路：路由删除后立即从路由表消失，更新周期未到，跳数可以无限累加所导致形成环路。
4）避环机制：

  • 抑制定时器hold time：无效路由不会立刻删除，0-180s视为无效，180s-240s挂起，到达240s后删除此条目。
  • 触发更新：以有变化，立刻发送更新。
  • 水平分割：接口接受的路由更新，绝对不可以反发回去
  • 毒性逆转： 无效路由通过触发更新被发送，跳数设置为16，16hop的更新意味着接受者删除该条目。
  • 最大跳数：15，无效跳数16
    5）RIPv1不支持VLSM，并不是说不能把经过VLSM的网络当做更新发送，如果接收的更新对应的网络本地就有，则不学习。

5.RIPv2配置（全局配置模式下）
router rip
version 2
no auto-summary //关闭自动汇总
network
配置手动汇总（在路由更新的出接口上，先进入接口）
ip summary-address rip <汇总后网络> <掩码> //也可以汇总成子网
6.RIPv2详解
1）与RIPv1的不同点：

  • v2可以关闭自动汇总，开启手动汇总，汇总的目的是为了减少对方路由表条目数量，增加路由效率。
  • v2支持VLSM，更新时携带掩码
  • v2多播更新，目的ip：224.0.0.9
  • v2支持被动接口（只收更新，不发送更新）和单播更新（指定目标ip接收更新即指定邻居）
  • v2支持协议认证，认证通过才能更新
  • v2支持偏移列表offset-list，偏移更新就是路由器在更新接收、发送时额外增加的跳数。

2）认证详解

  • 明文认证的工作机制：明文认证是将口令当做字符串直接发送给对端，对端进行比较，一致则通过认证，否则不通过。
  • 密文md5认证：md5实际上就是哈希算法，通过比较哈希值判断秘钥口令是否一致。
  • 认证的关键组件：钥匙链key chain、钥匙ID key、钥匙的口令string
  • 明文认证的工作过程：双方永远从ID最小的key开始发送，并且比较他们的string，钥匙间的轮转如果没有配置，那么永远都只发送最小ID的key。
  • 密文认证的工作过程：钥匙ID和钥匙的string参与md5计算，ID和哈希值被发送出去进行比较。

明文认证的钥匙ID不需要一致，密文认证的钥匙ID必须一致，因为参与md5的计算

  • 钥匙的轮转： 按照一定的顺序或者时间，分别用钥匙链上的不同钥匙进行匹配对比。

7.RIP认证的配置

  • step1：创建钥匙链
    key chain <钥匙名> //创建钥匙链
    key //定义钥匙id
    key-string <口令> //设定口令
  • step2：接口下引用钥匙链（首先进入接口）
    ip rip authentication key-chain <钥匙名>
    [ip rip authentication mode md5] //启用md5认证
